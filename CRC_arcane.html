MODEL CLASSES
Event
Responsibilities
  • Store event data (name, description, date, location, cost)
  • Manage registration dates (start/end times)
  • Track event status (DRAFT, OPEN, CLOSED, DRAWN, COMPLETED)
  • Track registration/attendee count
  • Manage registration data
  • Store geolocation and poster image URL
  • Maintain min/max entrants
Collaborators
  • EventRepository
  • EventService
  • WaitingListEntry
  • Decision
  • Users
Users
Responsibilities
  • Store user data (ID, name, email, phone)
  • Maintain unique ID for authentication
  • Track user role (ENTRANT, ORGANIZER, ADMIN)
  • Store geolocation (optional)
  • Manage notification preferences
  • Maintain device ID
  • Maintain registered event IDs
Collaborators
  • UserRepository
  • UserService
  • EventService
  • WaitingListEntry
  • Decision
Decision
Responsibilities
  • Track decision status (PENDING, INVITED, ACCEPTED, DECLINED, CANCELLED)
  • Manage decision metadata (timestamp, user ID, event ID)
  • Reference waiting list entry
  • Reference user
Collaborators
  • DecisionRepository
  • EventService
  • Event
  • WaitingListEntry
  • Users
WaitingListEntry
Responsibilities
  • Store waiting list entry data

  • Store join location (optional)
  • Track invitation timestamp (optional)
  • Link to event via subcollection
